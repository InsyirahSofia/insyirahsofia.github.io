🧪 Labs

Lab 1 — UI

Compose UI & Layouts
Reflection
This lab introduced me to the declarative nature of Jetpack Compose. I learned how to structure composable functions and build a responsive user interface using basic layouts, modifiers, and Material Design components. Understanding the unidirectional data flow was a key takeaway.

Lab 2 — Interaction

State & User Input
Reflection
I explored state management in Compose by handling user interactions such as button clicks, text input, and checkbox toggles. This lab reinforced the concept of hoisting state and using callbacks to communicate between composables — a pattern I now use daily.

Lab 3 — Material Design

Theming & Components
Reflection
I applied Material 3 theming and components to create a polished, consistent UI. Customizing colour schemes and typography helped me understand how to align an app’s visual identity with its purpose. I also used surfaces, cards, and navigation bars to improve usability.

Lab 4 — Navigation

Multi-screen Flows
Reflection
This lab taught me how to implement navigation between screens using the Navigation Component with Compose. I learned to pass arguments between destinations and manage the back stack, which became the foundation for both Project 1 and Project 2.

Lab 5 — Room

Local Database
Reflection
I integrated Room Database to persist user data locally. Defining entities, DAOs, and using coroutines for database operations gave me confidence in building offline-first features. This lab was critical preparation for Project 2's persistence layer.

📁 Projects

📱 Project 1 — LifeTrack

Lab 4 + Working Features

Problem: Many people find it difficult to track their health patterns accurately without a digital tool, often resulting in inconsistent monitoring and a lack of reliable history regarding their physical well-being. This manual approach is time‑consuming and fails to provide clear data‑driven insights.

Lessons learned
Building LifeTrack from scratch taught me how to structure a multi-screen app with ViewModel and state hoisting. I learned to design a cohesive user experience while managing shared state across fragments. This project strengthened my understanding of the MVVM pattern and the importance of separating UI logic from business logic.
📂 Google Drive

📱 Project 2 — LifeTrack

Lab 5 + API + Sensors + Cloud

Problem: Many people struggle to track their daily health consistently. They forget to monitor their steps, menstrual cycles, or set health goals. My app solves this by providing an all‑in‑one health tracking solution that works both online and offline.

Lessons learned
Project 2 pushed me to integrate four pillars: Room Database, Firebase Firestore, Retrofit Web API, and hardware sensors (step counter). I learned how to design a resilient app that works offline and syncs to the cloud. Handling real‑time Firestore updates and sensor lifecycle management were the most valuable challenges I overcame.

🧭 Reflection & Learning Journey

From the very first Compose UI lab to the final sensor‑driven project, I have grown tremendously as a mobile developer. I started with basic UI components and gradually moved to state management, navigation, persistence, networking, and hardware integration. Each lab built on the previous one, and by the time I reached Project 2, I felt confident building a complete, production‑ready app with offline support and cloud sync.

The most valuable lesson I learned is that good software design is about more than just code — it is about understanding the user's context and building solutions that are intuitive, reliable, and accessible. I am proud of what I built and excited to apply these skills to real‑world health technology.
